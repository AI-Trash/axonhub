# AGENTS.md

## Start here

- AxonHub is a mixed-language repo, but the **canonical backend is the Rust workspace** under `apps/` and `crates/`.
- The Go tree under `cmd/axonhub`, `conf`, and `internal/server/**` is **historical/reference material**. Do not implement new backend work there unless the user explicitly asks for legacy Go changes.
- When backend behavior is being migrated, use the Go code as an oracle if needed, but make the actual change in Rust.

## Fast validation

- Rust workspace: `cargo test --workspace --locked`
- Frontend install/build/lint: `pnpm install`, `pnpm lint`, `pnpm build`
- Frontend e2e: `pnpm test:e2e`
- Use **pnpm**, not npm or yarn.

## Rust backend rules

- Start backend work in `apps/axonhub-server` and the crates under `crates/`.
- The Rust binary entrypoint is `apps/axonhub-server/src/main.rs`.
- Keep workspace metadata centralized: member crates should continue using `version.workspace = true`, `edition.workspace = true`, `rust-version.workspace = true`, and `license.workspace = true`.
- **When adding a Rust dependency, declare it in the root `Cargo.toml` under `[workspace.dependencies]` and consume it with `*.workspace = true` in member crates.** Do not introduce per-crate version drift.
- The supported Rust database target is **PostgreSQL**. Use `config.example.yml` as the config contract; do not suggest SQLite for the Rust backend.
- New DB migrations belong in `crates/axonhub-db-migration/src/`, follow the existing `mYYYYMMDD_NNNNNN_description.rs` style, and must be registered in `crates/axonhub-db-migration/src/lib.rs`.

## Frontend rules

- The frontend lives in `frontend/` and uses **Vite + React 19 + TanStack Router + TanStack Query + Tailwind CSS v4 + Radix UI + ParaglideJS**.
- Keep using the existing workspace package pattern: most frontend dependency versions are centralized in `pnpm-workspace.yaml` via `catalog:`.
- Routes are defined in `frontend/src/routes/**`.
- `frontend/src/routeTree.gen.ts` is **generated by TanStack Router**. Do not edit it by hand.
- Translations live in `frontend/messages/en.json` and `frontend/messages/zh-CN.json`.
- For UI copy, use the Paraglide pattern already used across the app: `import * as m from '@/paraglide/messages'` and call `m.key()` helpers instead of hardcoding strings when a translation key belongs in messages.
- `frontend/src/paraglide/**` is generated by `paraglideVitePlugin` from `frontend/project.inlang/settings.json` and may be absent until frontend build/dev runs. Do not hand-edit generated Paraglide output.
- Frontend linting/formatting uses **Oxlint** and **Oxfmt**, not ESLint/Prettier.
- Frontend dev proxy behavior is defined in `frontend/vite.config.ts`; `/admin` and `/v1` proxy to `VITE_API_URL` (default `http://localhost:8090`).
- Frontend tests use Playwright from `frontend/playwright.config.ts`; the suite depends on `setup.spec.ts`, and the backend URL defaults to `http://localhost:8099` via `AXONHUB_API_URL`.

## Agent behavior for this repo

- Prefer minimal, focused changes and preserve existing patterns.
- Do not hand-edit generated files when the source files are available.
- If a task touches both legacy Go and Rust, confirm whether the request is about migration/reference or the current production path; default to the Rust path.
- If a frontend change affects copy, routes, or navigation, check whether it also requires updates to `frontend/messages/*.json`, generated routes, or Playwright coverage.
